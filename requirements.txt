# Pin ONLY the RL stack — the packages that must agree with each other and
# that Colab does not already ship at a compatible version. Everything else
# (numpy, pandas, tensorboard, matplotlib, opencv) is left to whatever Colab
# preinstalled: pinning exact patch versions of those just picks fights with
# Colab's base image (pandas==2.2.2, numba's numpy<2.1, tensorboard~=2.20)
# and produces "dependency conflict" warnings for no benefit.
#
# ale-py >=0.9 bundles the Atari ROMs inside the wheel, so there is no
# AutoROM / accept-rom-license step and no shimmy dependency — ale-py
# registers ALE/* envs with Gymnasium directly (see config.py, which calls
# gym.register_envs(ale_py) once). Confirmed working on Colab: ale-py 0.12.0,
# Python 3.12, CUDA.
#
# Lower bounds, not exact pins, so pip can satisfy them against Colab's
# existing numpy/etc. instead of downgrading and warning.
stable-baselines3[extra]>=2.9.0
gymnasium[atari]>=1.1.0
ale-py>=0.12.0

# moviepy 1.x has the API play.py's VecVideoRecorder path expects; 2.x renamed
# things, so cap it. imageio-ffmpeg supplies the encoder moviepy shells out to.
moviepy>=1.0.3,<2.0
imageio-ffmpeg>=0.4.9
